# Roommate Harmony - Judging Criteria Alignment

When presenting Roommate Harmony to the hackathon judges, focus on how the project perfectly aligns with standard hackathon criteria:

## 1. Innovation / Originality
**How we fit:** While there are dozens of expense-splitting apps (Splitwise) and chore-tracking apps (Sweepy), there are almost no **Social Awareness** apps. We aren't just digitizing a chore wheel; we are using AI to analyze *patterns of behavior* to predict and mediate human conflict before it happens. This is a novel application of AI in the prop-tech / social-tech space.

## 2. Technical Execution
**How we fit:** 
- **Frontend:** Built a highly polished, responsive Single Page Application using React and Tailwind CSS. The UI features custom design tokens, micro-animations, and a professional "built-by-humans" aesthetic.
- **AI Logic:** We built a custom scoring algorithm (`stabilityEngine.js`) that weighs chore completion, expense equity, and participation momentum to generate a real-time 'Stability Score'.

## 3. Real-World Impact / Utility
**How we fit:** Roommate conflict is a universal problem. Over 30% of adults in major cities live with roommates. Broken leases and lost friendships cost real money and emotional toll. Our solution provides immediate, tangible relief by acting as a neutral third party that simply "shows the data".

## 4. UI/UX & Design
**How we fit:** We specifically designed the app to NOT feel like a spreadsheet. We used warm colors (warm-white, sage green, amber) instead of harsh corporate blues. We implemented "soft alerts" with language designed to de-escalate tension ("This is not an accusation — just a signal"). The UX is centered entirely around empathy and clarity.

## 5. Completeness
**How we fit:** The project includes a complete frontend, a well-defined backend schema architecture, an AI engine algorithm ready for deployment, and a full landing page that clearly articulates the value proposition. It is pitch-ready.
